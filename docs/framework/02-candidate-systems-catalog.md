# Candidate Complex Systems Catalog

**Project:** Periodic Table of Complexity  
**Document type:** Seed reference catalog  
**Version:** 0.1  
**Purpose:** Provide a broad starting list of candidate complex systems, emergence transitions, model systems, and control/boundary cases for eventual quantification across complexity metrics.

---

## 1. How to Use This Catalog

This document is a seed list, not a final taxonomy. It is intended to help populate an initial database of systems that can later be measured, classified, compared, clustered, and embedded into a broader “periodic table of complexity.”

Entries include several kinds of objects:

| Kind | Meaning |
|---|---|
| **Concrete system** | A particular real-world system, such as the Internet, the Earth climate system, or the global financial system. |
| **System class** | A recurring type of system, such as markets, cities, ecosystems, codebases, or supply chains. |
| **Emergence transition** | A historical or structural transition where a new level of organization appeared, such as cells, multicellularity, language, agriculture, or cities. |
| **Subsystem** | A nested system within a larger system, such as the immune system inside an organism or the transportation network inside a city. |
| **Model system** | A simplified mathematical or computational system useful for comparison, such as cellular automata, the Ising model, or the Schelling segregation model. |
| **Boundary/control case** | A simple, complicated, random, or borderline system included to help distinguish complexity from other forms of organization. |

A useful rule for this project: **preserve nesting instead of flattening it**. Many complex systems are both systems and components of larger systems.

Examples:

```text
cell -> tissue -> organ -> organism -> population -> ecosystem -> biosphere
person -> household -> neighborhood -> city -> nation -> civilization
firm -> supply chain -> market -> economy -> global trade system
developer -> team -> codebase -> platform -> software ecosystem -> internet
```

## 3. Foundational Morowitz-Style Emergence Anchors

Harold J. Morowitz’s *The Emergence of Everything: How the World Became Complex* is useful because it gives a chronological “cosmic-to-cultural” spine: matter, stars, elements, planets, metabolism, cells, multicellular life, nervous systems, animals, mammals, primates, hominids, tools, language, agriculture, cities, technology, reflection, and meaning-making.

For this project, these should be treated as **P0 baseline systems and emergence transitions**. They are not merely historical episodes; each marks a new level of organization, constraint, feedback, or symbolic capacity.

| # | Priority | Candidate system / emergence | Catalog note |
|---:|---|---|---|
| 1 | P0 | **Primordium / early universe** | Origin of matter, energy, spacetime, and physical law as modeled by cosmology. |
| 2 | P0 | **Nonuniform universe / large-scale cosmic structure** | Density fluctuations, cosmic web, gravitational structure formation. |
| 3 | P0 | **Stars / stellar evolution** | Gravity, fusion, stellar life cycles, nucleosynthesis. |
| 4 | P0 | **Elements / periodic table / chemistry** | Emergence of chemical variety from quantum and nuclear constraints. |

## 30. Notes on Scope

A comprehensive catalog should intentionally include:

- **Systems with clear boundaries**, such as cells, organisms, codebases, and machines.
- **Systems with fuzzy boundaries**, such as cultures, economies, cities, ecosystems, and scientific fields.
- **Systems that are designed**, such as power grids, software platforms, and legal systems.
- **Systems that evolved**, such as ecosystems, organisms, language, and institutions.
- **Systems that are reflexive**, such as markets, politics, media, AI ecosystems, and human belief systems.
- **Systems that are mostly physical**, such as turbulent fluids, earthquakes, and climate.
- **Systems that are mostly informational**, such as language, software, knowledge graphs, and scientific communities.
- **Systems that are partly normative or symbolic**, such as law, religion, philosophy, culture, and meaning-making systems.

The key is not to force everything into one kind of ontology too early. The first phase should preserve heterogeneity. The later metric work can reveal which distinctions are fundamental and which are surface-level domain labels.
